## Terrakkoとは？

TerrakkoはDiscordから簡単な操作でNekko CloudのVMインスタンスの作成・変更・削除が行える対話型のコンソールです．  
Terrakkoに予めユーザ名・パスワード・SSHキーを登録することで，Proxmox VEのWebコンソールにアクセスしなくても，VM作成からSSH接続までの一連の操作がDiscordとターミナルで完結します．  

## 事前要件

- SSH公開鍵の作成(パスワード認証を使用する場合は不要)
- Node.js 18以上
- TypeScript

## 開発環境のセットアップ

開発者向けの環境構築手順です。エンドユーザーの場合は、[使い方](#使い方)セクションをご覧ください。

### 依存関係のインストール

プロジェクトのルートディレクトリで以下のコマンドを実行します。

```bash
cd app
npm install
```

### 環境変数の設定

#### テスト環境
テスト用の環境変数は `.env.test` ファイルに設定されています。テストを実行する際は自動的に読み込まれます。

#### 本番環境・開発環境
`app/` ディレクトリに `.env` ファイルを作成し、以下の変数を設定してください。

```bash
# Domain name
DOMAIN=".proxmoxve.internal.nekko-lab.dev"

# Proxmox VE
PVE_HOST="10.0.128.103"
PVE_USER="terrakko-agent@pve"
PVE_PASS="your-password"
PVE_REGION=["dev-proxmox-mk", "dev-proxmox-ur"]
PVE_TEMP_ID=["90100", "90101"]
PVE_TOKEN="terrakko"
PVE_SECRET="your-token-secret"

# Discord Bot
DIS_TOKEN="your-discord-bot-token"
DIS_CHANNEL_ID=1297112224665698336
```

> **注意**: 環境変数の値は実際の環境に合わせて変更してください。特に `PVE_REGION` と `PVE_TEMP_ID` は JSON 配列形式で記述する必要があります。

### テストの実行

Terrakkoはdevcontainer環境でテストを実行できます。

#### 1. すべてのテストを実行

```bash
npm test
```

#### 2. ウォッチモードでテストを実行

ファイルの変更を監視して、自動的にテストを再実行します。

```bash
npm run test:watch
```

#### 3. カバレッジレポート付きでテストを実行

テストカバレッジを確認しながらテストを実行します。

```bash
npm run test:coverage
```

#### テスト環境について

- テストは `.env.test` ファイルの環境変数を自動的に読み込みます
- Jest + ts-jest を使用したTypeScriptテスト環境
- 現在のテストファイル:
  - `src/utils/Validator.test.ts`: バリデーション機能のテスト (33テスト)
  - `src/infrastructure/config/Config.test.ts`: 環境変数読み込みのテスト (11テスト)

## 使い方

以下は、Discord上でTerrakkoを使用してVMを管理する手順です。

### メニューを呼び出す

Terrakkoとのやり取りは、まずメインメニューを呼び出すことから始まります。

#### コマンド形式

以下のいずれかの方法でメニューを呼び出すことができます:

- `@terrakko !` (メンション + 空白 + 感嘆符)
- `trk!` (プレフィックス + 感嘆符)

#### 手順

1. Discord チャンネルでコマンドを送信
2. Terrakko が応答し、`Start` ボタンが表示されます
3. `Start` ボタンをクリックしてメインメニューを呼び出します

    ![image1.png](/images/image1.png)

> **重要**: 各操作のセッションは**180秒**（3分）です。セッションが切れた場合は、再度コマンドを送信してメニューを呼び出してください。

### ユーザ情報をTerrakkoに登録する

VMを作成する前に、まずユーザー情報を登録する必要があります。

#### 手順

1. メインメニューから `Configure your info` ボタンをクリック
2. 入力フォームが表示されるので、以下の情報を入力します:
   - **Username**: VM ログイン用のユーザー名
   - **Password**: VM ログイン用のパスワード
   - **SSH Public Key**: SSH 公開鍵（パスワード認証のみの場合は空欄可）

    ![image2.png](/images/image2.png)

3. 入力内容を確認し、`Yes` をクリックして保存

> **注意**: 登録した情報は後から変更できます。同じ手順で再度情報を入力してください。

### VMを作成する

ユーザー情報を登録した後、VMを作成できます。

#### 手順

1. メインメニューから `Create your VM` ボタンをクリック

2. **VM数を選択**
   作成したいVMの個数を選択します（1〜5個）

    ![image3.png](/images/image3.png)

3. **VM名を入力**
   各VMの名前を入力します。複数のVMを作成する場合は、それぞれに名前を付けてください。

    ![image4.png](/images/image4.png)

4. **作成内容を確認**
   作成されるVMの情報（VM名、ユーザー名、パスワード、SSH鍵）が表示されます。

    ![image5.png](/images/image5.png)

5. **確認して作成**
   - `Yes`: VMを作成します
   - `No`: キャンセルします

    ![image6.png](/images/image6.png)

6. VM作成が完了すると、成功メッセージが表示されます

> **制限**: 一度に作成できるVMの最大数は**5個**までです。
> **注意**: VM名は一意である必要があります。既存のVM名と重複している場合はエラーになります。

### VMの電源を操作する

作成したVMの情報を確認したり、電源操作（起動・停止・再起動）を行うことができます。

#### 手順

1. メインメニューから `Show VM info` ボタンをクリック

2. **VMを選択**
   自分が所有するVMの一覧が表示されるので、操作したいVMを選択します

    ![image7.png](/images/image7.png)

3. **VM情報の確認**
   選択したVMの詳細情報が表示されます

    | 項目      | 詳細                                          |
    | :-------- | :-------------------------------------------- |
    | VM Name   | VMの名前                                      |
    | VMID      | Proxmox VEが管理するVM固有のID                |
    | Region    | VMのリージョン (mk/ur/tu)                     |
    | Status    | VMの状態 (running/stopped)                    |
    | Host Name | Service Discoveryが提供するVM固有のドメイン名 |
    | IPv4/v6   | VMのIPアドレス                                |

    ![image8.png](/images/image8.png)

4. **電源操作を実行**
   VMの状態に応じて、以下のボタンが利用できます:

   | ボタン     | 動作                        | 利用可能な状態 |
   | :--------- | :-------------------------- | :------------- |
   | `Start`    | VMを起動する                | stopped        |
   | `Shutdown` | VMを正常にシャットダウンする | running        |
   | `Reboot`   | VMを再起動する              | running        |
   | `Stop`     | VMを強制的に停止する        | running        |

> **推奨**: VMを停止する際は、`Shutdown` の使用を推奨します。`Stop` は強制停止のため、データ損失のリスクがあります。
> **注意**: 電源操作は自分が所有するVMに対してのみ実行できます。  

### VMを削除する

不要になったVMを削除できます。

#### 手順

1. メインメニューから `Delete VM` ボタンをクリック

2. **削除するVMを選択**
   自分が所有するVMの一覧が表示されるので、削除したいVMを選択します

    ![image9.png](/images/image9.png)

3. **VM情報の確認**
   削除対象のVMの詳細情報が表示されます

    ![image8.png](/images/image8.png)

4. **削除を確認**
   最終確認が表示されます:
   - `Yes`: VMを削除します（この操作は取り消せません）
   - `No`: キャンセルします

    ![image10.png](/images/image10.png)

5. 削除が完了すると、成功メッセージが表示されます

> **警告**: VMを削除すると、そのVMに保存されているすべてのデータが失われます。この操作は取り消せません。
> **重要**: VMを削除する前に、VMを停止しておく必要があります。実行中のVMは削除できません。
> **注意**: 削除できるのは自分が所有するVMのみです。

## トラブルシューティング

### セッションタイムアウトが発生する
操作中にセッションが切れた場合は、`trk!` コマンドを再度送信してメニューを呼び出してください。

### VM名が重複しているとエラーが出る
既存のVM名と異なる名前を指定してください。VM名は一意である必要があります。

### VMが作成できない
- ユーザー情報（Username、Password、SSH Key）が正しく登録されているか確認してください
- Proxmox VEのリソース（ディスク容量、メモリ）が十分にあるか確認してください

### その他の問題
詳細なエラー情報については、管理者にお問い合わせください。
